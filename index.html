PART I // SECTION 01 // ARCHITECTURE

Client-Server Architecture & The Fallacy of "Magic AI Code"

Before an AI model writes a single line of code, an engineer must dictate the boundaries of execution. The most fundamental boundary in modern software development is the division between the Client and the Server.

Rule 01 Never let AI decide your architecture. The LLM does not know your infrastructure constraints, scaling costs, or data privacy boundaries unless explicitly defined in your context window.

The Separation of Concerns

The client requests resources; the server executes business logic, manages persistent state, and serves responses. When building full-stack applications with AI tools, models frequently blur these lines by injecting heavy data queries directly into client-side components if left unchecked.

JavaScript 
// Bad: AI fetching sensitive data directly in client
useEffect(() => {
  fetch('https://api.example.com/admin/settings')
    .then(res => res.json())
}, []);

This is a massive CORS and security leak. The admin endpoint is exposed to the client bundle, visible in network tabs, and exploitable by anyone who opens DevTools. AI will generate this pattern by default because it optimizes for what works, not what's secure.

The Correct Pattern

Every sensitive operation must route through a server layer. The client speaks to your API; your API speaks to the database. This is not optional — it is the architectural constraint that keeps your application from becoming a security incident.

TypeScript 
// Server: routes.ts — The only place admin logic lives
import {  Router  } from 'express';
import {  authenticateAdmin  } from './middleware/auth';

const router = Router();

router.get('/admin/settings',
  authenticateAdmin,  // Auth middleware — never skipped
  async (req, res) => {
    const settings = await db.query(
      'SELECT * FROM admin_settings WHERE org_id = $1',
      [req.user.orgId]
    );
    res.json(settings.rows);
  }
);
Senior Move When using AI to generate API routes, always include your auth middleware pattern in the prompt. AI will skip auth by default if you don't remind it. Treat security as a non-negotiable constraint in every generation request.

Key Takeaways

Concept Mental Model Senior Insight
Client-Server Request vs. Execution boundary Never expose sensitive endpoints to the client bundle
CORS Browser-enforced trust boundary It protects users, not your server — validate on both sides
Auth Every request is untrusted until proven otherwise AI skips auth unless you explicitly include it in prompts
