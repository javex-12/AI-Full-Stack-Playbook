## The AI Application Stack

Building an AI app isn't just calling an API. It's architecting a system that handles prompts, manages context, streams responses, handles errors, and scales under load.

```
Frontend (React) → Backend (Node) → AI API (Groq/OpenAI)
                        │
                        ▼
                  Database (history)
```

## Step 2: System Prompt Architecture

The system prompt is the most important part of your AI app. It defines the model's behavior, constraints, and output format.

### The Anatomy

```
┌─────────────────────────────────────────┐
│ 1. ROLE                                  │
│    Who the model is acting as            │
├─────────────────────────────────────────┤
│ 2. CONTEXT                               │
│    Background information                │
├─────────────────────────────────────────┤
│ 3. INSTRUCTIONS                          │
│    Rules and constraints                 │
├─────────────────────────────────────────┤
│ 4. OUTPUT FORMAT                         │
│    How to structure the response         │
├─────────────────────────────────────────┤
│ 5. EXAMPLES (optional)                   │
│    Input/output demonstrations           │
└─────────────────────────────────────────┘
```

### Example: Customer Support Bot

```typescript
const SYSTEM_PROMPT = `
<role>
You are a customer support agent for "TechStore", an online electronics retailer.
You are helpful, professional, and solution-oriented.
</role>

<context>
TechStore sells laptops, phones, tablets, and accessories.
Orders can be tracked at track.techstore.com.
Return policy: 30 days from delivery, item must be in original packaging.
Refunds process within 5-7 business days.
</context>

<instructions>
- Always greet the customer by name if provided
- If the issue is about order status, direct them to track.techstore.com
- If the issue is about returns, confirm the order is within 30 days
- If you cannot resolve the issue, offer to connect them with a human agent
- Never make up information about products, prices, or policies
- Never share other customers' information
- Keep responses under 150 words
</instructions>

<output_format>
Respond in a friendly, professional tone.
Use bullet points for multiple steps.
Always end with: "Is there anything else I can help with?"
</output_format>
`;
```

## Step 3: API Integration

### Groq Integration (Fastest)

```typescript
// Using the Groq SDK
import Groq from 'groq-sdk';

const groq = new Groq({
  apiKey: process.env.GROQ_API_KEY,
});

// Basic completion
async function chat(userMessage: string, history: Message[]) {
  const response = await groq.chat.completions.create({
    model: 'llama-3.3-70b-versatile',
    messages: [
      { role: 'system', content: SYSTEM_PROMPT },
      ...history,
      { role: 'user', content: userMessage },
    ],
    temperature: 0.7,
    max_tokens: 1024,
  });

  return response.choices[0].message.content;
}
```

### Streaming Response

```typescript
// Streaming — tokens arrive as they're generated
async function chatStream(userMessage: string, history: Message[]) {
  const stream = await groq.chat.completions.create({
    model: 'llama-3.3-70b-versatile',
    messages: [
      { role: 'system', content: SYSTEM_PROMPT },
      ...history,
      { role: 'user', content: userMessage },
    ],
    stream: true,  // Enable streaming
  });

  // Process tokens as they arrive
  for await (const chunk of stream) {
    const token = chunk.choices[0]?.delta?.content || '';
    process.stdout.write(token);  // Send to client immediately
  }
}
```

## Step 4: Frontend Integration

### React Chat Component

```tsx
// components/Chat.tsx
import { useState, useRef, useEffect } from 'react';

interface Message {
  role: 'user' | 'assistant';
  content: string;
}

export function Chat() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [input, setInput] = useState('');
  const [isLoading, setIsLoading] = useState(false);
  const messagesEndRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' });
  }, [messages]);

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    if (!input.trim() || isLoading) return;

    const userMessage = { role: 'user' as const, content: input };
    setMessages(prev => [...prev, userMessage]);
    setInput('');
    setIsLoading(true);

    try {
      const response = await fetch('/api/chat', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          message: input,
          history: messages,
        }),
      });

      const reader = response.body?.getReader();
      const decoder = new TextDecoder();

      // Add empty assistant message
      setMessages(prev => [...prev, { role: 'assistant', content: '' }]);

      // Stream tokens into the message
      while (true) {
        const { done, value } = await reader!.read();
        if (done) break;

        const token = decoder.decode(value);
        setMessages(prev => {
          const last = prev[prev.length - 1];
          return [...prev.slice(0, -1), { ...last, content: last.content + token }];
        });
      }
    } catch (err) {
      console.error('Chat error:', err);
    } finally {
      setIsLoading(false);
    }
  }

  return (
    <div className="chat-container">
      <div className="messages">
        {messages.map((msg, i) => (
          <div key={i} className={`message ${msg.role}`}>
            {msg.content}
          </div>
        ))}
        <div ref={messagesEndRef} />
      </div>

      <form onSubmit={handleSubmit}>
        <input
          type="text"
          value={input}
          onChange={(e) => setInput(e.target.value)}
          placeholder="Type your message..."
          disabled={isLoading}
        />
        <button type="submit" disabled={isLoading}>
          {isLoading ? 'Thinking...' : 'Send'}
        </button>
      </form>
    </div>
  );
}
```

## Step 5: Backend API Route

### Next.js API Route

```typescript
// app/api/chat/route.ts
import { Groq } from 'groq-sdk';

const groq = new Groq({
  apiKey: process.env.GROQ_API_KEY,
});

const SYSTEM_PROMPT = `Your system prompt here`;

export async function POST(req: Request) {
  const { message, history } = await req.json();

  // Validate input
  if (!message || typeof message !== 'string') {
    return new Response('Invalid message', { status: 400 });
  }

  // Rate limiting check
  // (implement with Redis or in-memory store)

  const stream = await groq.chat.completions.create({
    model: 'llama-3.3-70b-versatile',
    messages: [
      { role: 'system', content: SYSTEM_PROMPT },
      ...history.slice(-10),  // Keep last 10 messages for context
      { role: 'user', content: message },
    ],
    stream: true,
    temperature: 0.7,
    max_tokens: 1024,
  });

  // Convert to ReadableStream for client
  const encoder = new TextEncoder();
  const readable = new ReadableStream({
    async start(controller) {
      for await (const chunk of stream) {
        const token = chunk.choices[0]?.delta?.content || '';
        controller.enqueue(encoder.encode(token));
      }
      controller.close();
    },
  });

  return new Response(readable, {
    headers: {
      'Content-Type': 'text/plain; charset=utf-8',
      'Cache-Control': 'no-cache',
    },
  });
}
```

## Step 6: Add Conversation History

Store chat history so users can continue conversations.

```typescript
// Using a simple database (PostgreSQL)
CREATE TABLE chat_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  title TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE chat_messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID REFERENCES chat_sessions(id),
  role TEXT NOT NULL CHECK (role IN ('user', 'assistant', 'system')),
  content TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Load last 20 messages for a session
SELECT role, content
FROM chat_messages
WHERE session_id = $1
ORDER BY created_at DESC
LIMIT 20;
```

## Step 7: Add Function Calling

Let the AI call your functions when needed.

```typescript
const tools = [
  {
    type: 'function',
    function: {
      name: 'get_order_status',
      description: 'Check the status of a customer order',
      parameters: {
        type: 'object',
        properties: {
          orderId: { type: 'string', description: 'The order ID' },
        },
        required: ['orderId'],
      },
    },
  },
];

// In your API route
const response = await groq.chat.completions.create({
  model: 'llama-3.3-70b-versatile',
  messages: [...],
  tools,
  tool_choice: 'auto',
});

// Handle tool calls
if (response.choices[0].message.tool_calls) {
  const toolCall = response.choices[0].message.tool_calls[0];
  const args = JSON.parse(toolCall.function.arguments);

  // Execute the function
  const result = await getOrderStatus(args.orderId);

  // Send result back to the model
  messages.push(response.choices[0].message);
  messages.push({
    role: 'tool',
    tool_call_id: toolCall.id,
    content: JSON.stringify(result),
  });

  // Get final response with the tool result
  const finalResponse = await groq.chat.completions.create({
    model: 'llama-3.3-70b-versatile',
    messages,
  });
}
```

## Common Pitfalls

| Pitfall | Why It Happens | Fix |
|---------|---------------|-----|
| **Model hallucinates facts** | No context provided | Add RAG with your own data |
| **Response is too long** | No token limit | Set `max_tokens` |
| **Model ignores instructions** | System prompt is vague | Use XML delimiters, be specific |
| **API calls are slow** | Using wrong model | Use Groq for speed, GPT-4o for quality |
| **Costs spiral** | No caching | Cache common responses in Redis |
| **Rate limiting** | No protection | Add rate limiting on your API |
