## Project-Based Learning

The best way to learn is to build. Here are 10 projects organized by difficulty — each one teaches specific skills from the playbook.

---

## Beginner Projects (1-2 Days Each)

### Project 1: Personal Portfolio

**What you build:** A one-page portfolio showcasing your work.

**Skills practiced:**
- Static site generation (Astro)
- Responsive design (CSS Grid, Flexbox)
- Typography and whitespace (Part X)
- SEO basics (meta tags, OG images)

**Features:**
- Hero section with name and tagline
- Project gallery (3-5 projects with images)
- About section with skills
- Contact form (Formspree)
- Dark mode toggle

**Tech stack:** Astro + Tailwind CSS

---

### Project 2: Markdown Blog

**What you build:** A blog that reads markdown files and renders them as pages.

**Skills practiced:**
- Content collections (Astro)
- Dynamic routing
- MDX syntax
- Syntax highlighting
- RSS feed generation

**Features:**
- List all posts on homepage
- Individual post pages with table of contents
- Code syntax highlighting
- Estimated reading time
- RSS feed

**Tech stack:** Astro + MDX + Shiki

---

### Project 3: Landing Page

**What you build:** A high-converting landing page for a fictional product.

**Skills practiced:**
- Editorial design (Part X)
- CSS animations (GSAP or CSS keyframes)
- Performance optimization
- A/B testing setup

**Features:**
- Hero with animated text reveal
- Feature grid with icons
- Pricing table
- Testimonials section
- CTA with email capture

**Tech stack:** Astro + GSAP + Tailwind

---

## Intermediate Projects (3-5 Days Each)

### Project 4: AI Chatbot

**What you build:** A chat interface that talks to an LLM API.

**Skills practiced:**
- API integration (Groq/OpenAI)
- Streaming responses
- System prompt engineering (Part XIV)
- State management
- Error handling

**Features:**
- Real-time streaming responses
- Conversation history
- Code syntax highlighting in responses
- Copy message button
- Model selection (GPT-4o, Claude, Llama)

**Tech stack:** Next.js + Groq API + React

---

### Project 5: Task Manager (SaaS)

**What you build:** A Kanban-style task manager with user accounts.

**Skills practiced:**
- Authentication (Part VII)
- Database design (Part VI)
- CRUD operations
- Real-time updates (WebSockets)
- Drag-and-drop UI

**Features:**
- User registration/login (Supabase Auth)
- Create, edit, delete tasks
- Drag-and-drop between columns (To Do, In Progress, Done)
- Due dates and priority levels
- Team collaboration (shared boards)

**Tech stack:** Next.js + Supabase + Tailwind

---

### Project 6: E-commerce Store

**What you build:** A product catalog with cart and checkout.

**Skills practiced:**
- Payment integration (Stripe)
- Inventory management
- Order processing
- Email notifications
- SEO (product pages, structured data)

**Features:**
- Product listing with filters
- Individual product pages
- Shopping cart (persistent)
- Stripe checkout
- Order confirmation emails
- Admin dashboard

**Tech stack:** Next.js + Stripe + Supabase

---

## Advanced Projects (1-2 Weeks Each)

### Project 7: AI Content Generator

**What you build:** An AI-powered tool that generates blog posts, social media content, or marketing copy.

**Skills practiced:**
- RAG (Retrieval-Augmented Generation)
- Prompt templates
- Token management
- Usage tracking
- Subscription billing

**Features:**
- Multiple content types (blog, tweet, email)
- Custom tone and style settings
- Content history and favorites
- Export to markdown/PDF
- Usage-based pricing (Stripe)

**Tech stack:** Next.js + Groq + Supabase + Stripe

---

### Project 8: Real-time Dashboard

**What you build:** A analytics dashboard that updates in real-time.

**Skills practiced:**
- Data visualization (Chart.js, Recharts)
- WebSocket connections
- Server-Sent Events
- Caching strategies (Redis)
- Performance optimization

**Features:**
- Live charts that update without refresh
- Multiple data sources
- Date range picker
- Export to CSV
- Responsive layout (works on mobile)

**Tech stack:** Next.js + Redis + WebSockets + Recharts

---

### Project 9: Developer Tools API

**What you build:** A public API that developers can use (like a URL shortener, image optimizer, or code formatter).

**Skills practiced:**
- API design (RESTful endpoints)
- Rate limiting
- API key authentication
- Documentation (OpenAPI/Swagger)
- Monitoring and logging

**Features:**
- API key registration
- Usage dashboard
- Rate limiting per key
- Interactive API docs
- Webhook support

**Tech stack:** Node.js + Express + Redis + PostgreSQL

---

### Project 10: Full-Stack SaaS

**What you build:** A complete SaaS application with authentication, billing, and multi-tenancy.

**Skills practiced:**
- Everything from the playbook combined
- Multi-tenant architecture
- Subscription management
- Role-based access control
- CI/CD pipeline
- Production monitoring

**Features:**
- User authentication (OAuth + email)
- Organization/team management
- Role-based permissions (admin, editor, viewer)
- Stripe subscription billing
- Usage analytics
- Admin dashboard
- Email notifications
- API access

**Tech stack:** Next.js + Supabase + Stripe + Redis + Vercel

## How to Use These Projects

1. **Start with Project 1** — even if you're experienced. It warms you up.
2. **Build each project fully** — don't skip steps, don't use placeholders.
3. **Deploy every project** — a project on Vercel is worth 10x more than one on your laptop.
4. **Add to your portfolio** — each project becomes a showcase piece.
5. **Write about it** — blog posts about what you built = SEO + learning reinforcement.
